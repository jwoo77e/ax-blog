## 대표님이 말했다: "AI 비서 만들어줘"

어느 날 대표님이 말했다. "임직원들한테 AI 비서 만들어줘라."

리스크제로는 약 50명 규모의 산업안전 기업이다. 대표님의 지시는 명쾌했다. 근로자 안전을 AI로 지키는 회사가, 정작 내부 직원들은 AI를 안 쓰고 있으니 말이 안 된다는 거다. 맞는 말이다.

그런데 ChatGPT 계정 50개 사주는 게 답일까? 월 $20 × 50명 = $1,000. 비용만의 문제가 아니다. 사내 규정을 모르는 AI, 우리 프로젝트 맥락을 모르는 AI는 그냥 고급 검색엔진이다. "AI 비서"라 부르기엔 민망한 수준.

그래서 직접 만들기로 했다. 그리고 지금, 그 현실적인 여정을 기록한다.

## 요구사항을 정리하니 만만치 않다

팀별로 뭘 원하는지 정리해봤다.

- **사내 문서 기반 답변**: 안전관리 규정, 프로젝트 문서, 사내 공지를 학습한 RAG 기반 응답
- **회의록 자동 생성**: 회의 녹음 넣으면 요약과 액션아이템까지 뽑아주는 것
- **개인별 맞춤**: 안전컨설팅그룹과 개발팀이 같은 AI를 쓸 수는 없다. 부서별, 역할별 다른 컨텍스트가 필요하다
- **보안**: 사내 데이터가 외부로 나가면 안 된다. 대표님의 첫 번째 질문은 항상 "그거 데이터 어디로 가?"다
- **비용**: 스타트업이다. 월 $1,000도 아까운데, 더 싸게 할 방법은?

한 줄 요약: **싸고, 안전하고, 똑똑하고, 개인화된 AI 비서**. 세상에 그런 게 있으면 다들 쓰고 있겠지.

## 검토한 4가지 스택

현실적으로 검토한 조합은 네 가지다.

**① OpenClaw + Gemma4 (로컬 LLM)**
서버에 Gemma4를 올리고 OpenClaw로 에이전트 인터페이스를 씌우는 방식. 데이터가 서버 밖으로 안 나가니 보안은 최고. 비용도 서버 유지비뿐이다. 문제는 성능. Gemma4는 범용 대화에서는 괜찮지만, 복잡한 추론이나 한국어 문서 요약에서 할루시네이션이 꽤 나온다. "그럴듯하게 틀리는" AI 비서는 위험하다.

**② OpenClaw + Claude API**
성능은 확실하다. Claude는 긴 문서 요약, 한국어 처리, 복잡한 지시 수행에서 체감이 다르다. 하지만 API 비용이 사용량에 비례해서 올라가고, 데이터가 외부 서버를 거친다. 대표님의 보안 레이더에 즉시 걸린다.

**③ 자체 개발 — RZ Agent Plus**
Node.js + OpenAI Agents SDK 기반으로 에이전트 19개를 직접 짰다. 회의록 에이전트, 규정 검색 에이전트, 보고서 초안 에이전트 등. 가장 유연하고 우리 업무에 딱 맞출 수 있다. 대신 개발 공수가 어마어마하다. 50명 회사에서 이걸 유지보수할 인력이 나인데, 나는 한 명이다.

**④ SaaS — Zeromate (ai.riskzero.ai)**
사실 이미 만들어서 운영 중이다. 56명이 등록했다. 그런데 활성 사용자가 6명이다. 전체의 11%. 만들었는데 안 쓴다. 이게 가장 뼈아픈 현실이다.

## 현실의 벽들

기술적 문제보다 사람의 문제가 더 컸다.

**할루시네이션**: Gemma4로 사내 규정을 물어보면 가끔 존재하지 않는 조항을 만들어낸다. 안전 분야에서 이건 치명적이다. "산업안전보건법 제XX조에 따르면..."이라고 AI가 자신 있게 말하는데, 그 조항이 없다면? 신뢰가 한 번에 무너진다.

**보안 vs 성능 딜레마**: 로컬 모델은 안전하지만 멍청하고, 클라우드 모델은 똑똑하지만 대표님이 허락을 안 한다. 이 사이에서 균형점을 찾는 게 지금 가장 큰 숙제다.

**활성 사용자 11%의 진실**: 56명 중 6명만 쓴다. 나머지 50명이 게으른 게 아니다. 기존 업무 방식이 익숙한 사람에게 "AI한테 물어봐"는 오히려 귀찮은 일이다. 습관을 바꾸는 건 시스템을 바꾸는 것보다 훨씬 어렵다.

**회의록 품질**: Gemini API 기반으로 회의록 자동 생성을 시도했는데, 전문 용어가 많은 산업안전 회의에서는 정확도가 아쉬웠다. Claude로 전환을 검토 중이지만, 다시 비용과 보안 문제로 돌아온다.

## 지금은 이렇게 하고 있다

완벽한 답을 찾은 건 아니다. 지금은 병렬로 여러 실험을 돌리고 있다.

- **OpenClaw + Gemma4 PoC**: 내가 직접 테스트 대상이다. 매일 업무에 로컬 LLM을 끼워넣으며 어디까지 쓸 수 있는지 한계를 탐색 중
- **Genspark Claw를 메인 비서로 활용**: Claude 기반 에이전트를 내 개인 실무에 전면 투입했다. 트렌드 분석, 보고서 초안, 특허 명세서 검토까지. 사실상 이게 지금 "AI 비서"의 완성형에 가장 가깝다
- **사용 현황 모니터링**: ai.riskzero.ai에 Playwright 스크래퍼를 붙여서 누가 얼마나 쓰는지 대시보드로 추적한다. 데이터 없이는 개선도 없으니까

## 여기까지 와서 배운 것

**"AI 비서"는 기술의 문제가 아니라 문화의 문제다.**
최신 모델을 붙여도 안 쓸 사람은 안 쓴다. 반대로, 좀 부족한 모델이라도 업무 흐름에 자연스럽게 녹아들면 쓴다. 기술 선택보다 "어떻게 쓰게 만들 것인가"가 핵심이다.

**최고의 모델이 아니라 "쓸 수 있는" 모델이 답이다.**
GPT-4o가 좋은 건 안다. Claude가 뛰어난 것도 안다. 하지만 보안 정책, 비용 한도, 인프라 제약 안에서 실제로 돌릴 수 있는 모델이 진짜 답이다. 완벽을 기다리면 영원히 시작 못 한다.

**톱다운보다 바텀업이 효과적이다.**
대표님이 "AI 써라"고 지시해도 안 쓰는 사람은 안 쓴다. 그런데 옆자리 동료가 AI로 30분짜리 보고서를 5분 만에 뽑아내는 걸 보면? 그때 물어본다. "그거 어떻게 하는 거야?" 한 명이 먼저 잘 쓰는 모습을 보여주는 게 50명짜리 교육보다 낫다.

지금 그 "한 명"이 되려고 매일 삽질 중이다. 완성되면 자랑하겠다. 안 되면 그것도 솔직히 쓰겠다.
