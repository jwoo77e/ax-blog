매일 아침 출근해서 맥북을 열면, Slack에 메시지가 와 있다. AI가 보낸 거다.

오늘의 AI 뉴스 요약, 산업안전 분야 동향, OpenAI와 Anthropic의 최신 업데이트, 하네스 엔지니어링 트렌드. 그리고 IRIS에 새로 올라온 연구과제 공고까지. 커피 한 잔 마시면서 훑어보면 그날의 기술 동향 파악이 끝난다.

이걸 만들기까지의 삽질기를 적어본다.

## 모닝 브리핑 시스템

출발점은 단순했다. 나는 매일 아침 30분 넘게 뉴스를 뒤졌다. TechCrunch, Hacker News, OpenAI 블로그, Anthropic 블로그, 그리고 "개인 AI Agent" 관련 글들. 이걸 빼먹으면 왠지 불안했다. AI 분야가 하루가 다르게 바뀌니까.

그래서 OpenClaw 크론잡으로 자동화했다. 매일 오전 8시에 실행되는 브리핑 작업이다.

수집하는 항목은 7가지:

1. 개인 AI Agent 동향
2. 하네스 엔지니어링 (AI orchestration)
3. 산업안전 AI 적용 사례
4. TechCrunch AI 뉴스
5. Hacker News 인기 글
6. OpenAI 공식 뉴스
7. Anthropic 공식 뉴스

동작 방식은 이렇다. `gsk search`로 각 주제를 웹 검색하고, 유의미한 결과가 있으면 `gsk crawl`로 원문을 긁어온 뒤 핵심만 요약한다. 그 결과를 Slack DM으로 보내준다.

```
[크론잡 흐름]
08:00 트리거
  → gsk search "personal AI agent news today"
  → gsk search "industrial safety AI 2026"
  → gsk search "OpenAI latest update"
  → ... (7개 주제)
  → 각 결과 요약 생성
  → Slack DM으로 전달
```

처음 실행했을 때 바로 에러가 났다.

```
Error: Delivering to Slack requires target
```

크론잡은 대화 맥락이 없어서, 결과를 "누구에게" 보낼지 모르는 거였다. Slack 유저 ID를 직접 지정해서 해결했다.

```
target: user:U08V76EJ7CZ
```

이 한 줄 추가하는 데 반나절 걸렸다. 크론잡 디버깅은 실행 → 기다리기 → 로그 확인의 반복이라 피드백 루프가 길다.

## IRIS 연구과제 크롤러

모닝 브리핑이 안정되니까 욕심이 생겼다. 연구과제 공고도 자동화하자.

리스크제로는 R&D 과제를 꾸준히 수주한다. 그런데 IRIS(국가연구개발사업 통합정보시스템)에서 공고를 매번 수동으로 검색하는 건 번거로운 일이다. 키워드도 여러 개고, 부처도 여러 곳이니까.

매일 오전 9시에 crontab으로 돌리는 Python 스크립트를 만들었다.

```python
# 검색 조건
keywords = ["AI", "산업안전", "스마트", "안전보건", "위험"]
statuses = ["접수예정", "접수중"]
ministries = ["과학기술정보통신부", "중소벤처기업부", 
              "행정안전부", "고용노동부"]
```

IRIS API를 직접 호출해서 5개 키워드 × 2개 상태 × 4개 부처 조합으로 공고를 수집한다. 중복 제거하고, D-day를 계산해서 마감 임박 순으로 정렬한다. 결과는 엑셀 파일로 저장하고, Gmail로 자동 전송한다.

이 크롤러도 삽질의 연속이었다.

**첫 번째 문제: URL 404.** IRIS 웹사이트에서 개발자 도구를 열어 네트워크 탭을 보니, 화면에 보이는 URL과 실제 API 엔드포인트가 달랐다. JavaScript를 분석해서 진짜 호출 주소를 찾아냈다.

**두 번째 문제: 이메일 전송.** 처음에 `gsk email`로 보내려 했는데 동작하지 않았다. 결국 `gog gmail send`로 전환했더니 깔끔하게 해결됐다.

**세 번째 문제: 데이터 품질.** 초기 버전은 공고 제목과 마감일만 가져왔는데, 쓸모가 없었다. 공고 요약, 첨부파일 목록, HTML 엔티티 정리(&amp; → & 같은 변환)를 추가하면서 실제로 쓸 만한 수준이 됐다.

## 주간 자동화

일일 작업이 안정되니까 주간 단위도 자동화했다.

- **주간 리뷰**: 매주 금요일 16:00 — 이번 주 작업 정리, AI 트렌드 요약, 다음 주 계획
- **주간 목표 검토**: 매주 월요일 09:00 — 이번 주 목표 리마인더

주간 리뷰에서 또 한 번 삽질했다. 처음에 타임아웃을 3분으로 잡았는데, 웹 검색 7개 주제 + 분석 + 요약을 3분 안에 끝내는 건 무리였다. 실행 중간에 잘려서 반쪽짜리 리뷰가 왔다. 타임아웃을 10분으로 늘리니까 해결됐다.

```yaml
# 주간 리뷰 크론잡 설정
schedule: "0 16 * * 5"  # 매주 금요일 16:00
timeout: 600             # 10분 (처음 180초 → 실패)
```

교훈: AI 작업의 실행 시간은 넉넉하게 잡아야 한다. 웹 검색이 들어가면 특히 그렇다.

## 삽질 요약

| 문제 | 원인 | 해결 |
|------|------|------|
| Slack 전달 안 됨 | 크론잡에 타겟 미설정 | `user:U08V76EJ7CZ` 지정 |
| 주간 리뷰 중간 절단 | 타임아웃 3분 | 10분으로 확장 |
| IRIS API 404 | 웹 URL ≠ API URL | JS 분석으로 엔드포인트 발견 |
| 이메일 전송 실패 | gsk email 미지원 | gog gmail send로 전환 |

자동화 세팅의 80%는 삽질이고, 20%가 실제 로직이다. 하지만 한 번 잡아놓으면 그 뒤로는 신경 쓸 일이 없다.

## 체감 효과

지금 이 시스템이 매일 돌아간 지 2주 정도 됐다.

**시간 절약**: 하루 30분 이상. 아침마다 여러 사이트 돌아다니며 뉴스 확인하던 시간이 사라졌다. Slack 메시지 하나로 끝난다.

**공고 누락 제로**: 예전에는 IRIS를 일주일에 한두 번 들어가서 확인했다. 지금은 매일 수집되니까 마감 D-3인 공고를 놓칠 일이 없다. 엑셀에 D-day가 찍혀 있어서 긴급한 것부터 확인하면 된다.

**불안감 해소**: "혹시 중요한 AI 뉴스 놓친 거 아닌가?" 하는 FOMO가 사라졌다. AI가 알아서 걸러주니까.

## 자동화의 핵심

크론잡 자동화의 핵심은 **"한 번 세팅하고 잊어버리는 것"**이다.

매일 수동으로 하던 일을 자동화하면, 처음 며칠은 "잘 돌아가나?" 확인하게 된다. 그런데 일주일쯤 지나면 진짜로 잊어버린다. 아침에 출근하면 Slack에 브리핑이 와 있고, 메일함에 연구과제 목록이 들어와 있다. 그게 당연해진다.

그 "당연함"을 만드는 게 자동화의 가치다. 거창한 시스템이 아니다. 크론잡 하나, 스크립트 하나. 그걸로 하루 30분을 벌고, 놓치는 정보를 없앤다.

다음 글에서는 이런 자동화를 넘어서, AI에게 실제 업무를 위임하는 이야기를 해볼 생각이다. 특허 명세서 초안, 보고서 초안 같은 것들. 수집이 아니라 생산을 맡기는 단계다.
