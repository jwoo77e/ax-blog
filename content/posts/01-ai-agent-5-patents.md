## 발단: "특허 5건을 만들어야 한다"

회사의 핵심 기술을 특허로 보호하려면 하나의 거대한 특허가 아니라, 기술 요소별로 쪼개서 출원하는 게 전략적으로 유리하다.

문제는 시간이다. 특허 명세서 1건 쓰는 데 보통 며칠이 걸린다. 5건이면? 변리사 의뢰하면 몇 주, 직접 쓰면 한 달.

그래서 AI에게 시켰다.

## 5건의 특허 구성

기존에 보유한 기술 문서들을 먼저 AI에게 분석시켰다. 핵심 알고리즘의 기술 상세를 파악한 후, 서로 다른 기술 측면에서 5개의 특허 주제를 도출했다. 각 특허는 독립적인 기술 요소를 다루되, 전체적으로는 하나의 시스템을 구성하는 구조다.

## 병렬 실행: "5개를 동시에"

핵심은 **병렬 실행**이었다. 5건의 특허는 서로 독립적이다 — 1번이 끝나야 2번을 시작하는 게 아니라, 동시에 진행할 수 있다.

AI 에이전트 시스템의 의존성 맵을 확인하고, 5개의 서브에이전트를 동시에 생성했다:

```
Master Agent ──┬→ spawn(특허1)
               ├→ spawn(특허2)
               ├→ spawn(특허3)
               ├→ spawn(특허4)
               └→ spawn(특허5)
               
→ sessions_yield() → 결과 수신 → 통합 보고
```

## 결과: 약 15분

5개의 AI 인스턴스가 동시에 작업을 시작했다. 대시보드에서 5개 에이전트가 모두 "🔄 작업 중"으로 표시되는 걸 보는 건 꽤 인상적이었다.

가장 먼저 끝난 건 약 8분, 가장 오래 걸린 건 약 13분. 전체 5건이 완료되기까지 **약 15분**.

각 특허마다 Word 문서 + 도면 5~7장 + 청구항 10~12개가 생성됐다. 총 **27장 도면, 52개 청구항, Word(.docx) 5건**. 전수 검증 결과 5건 모두 PASS.

## 완벽하진 않았다

솔직히 말하면, 첫 결과물이 바로 완벽하진 않았다.

- **도면 잘림**: 이미지 높이가 A4 페이지를 초과해서 하단이 잘린 것. 최대 높이를 4.5인치로 줄이고 각 도면을 독립 페이지에 배치해서 해결했다.
- **화살표 겹침**: matplotlib으로 그린 블록 다이어그램에서 화살표 경로가 블록과 겹치는 문제. 레이아웃을 재설계해서 수정했다.
- **한글 폰트 깨짐**: `AppleGothic` 대신 `Apple SD Gothic Neo`로 변경하니 해결.

이런 수정 과정이 2~3번 반복됐다. 그래도 사람이 직접 쓰는 것보다는 훨씬 빠르다.

## 배운 것

### 1. 병렬 실행의 위력
순차로 했으면 5건 × 13분 = 65분. 병렬로 하니 15분. **4배 빠르다.**

### 2. 검증 단계가 핵심
AI가 만든 산출물을 그냥 쓰면 안 된다. 도면 잘림, 폰트 깨짐, 화살표 겹침 — 이런 건 자동 검증으로 1차 필터링하고, 사람이 2차 확인해야 한다. Martin Fowler가 말한 **Feedback 센서**가 바로 이거다.

### 3. "초안"의 가치
이 5건은 변리사에게 바로 제출할 수준은 아니다. 하지만 **전체 구조와 핵심 내용이 잡힌 초안**으로서 가치가 크다. 변리사가 백지에서 시작하는 것과, 이 초안을 다듬는 것은 공수가 완전히 다르다.

### 4. 하네스가 중요하다
5개 에이전트가 동시에 돌아갈 수 있었던 건, **하네스(Harness)**가 갖춰져 있었기 때문이다. 라우팅 규칙, 의존성 맵, Feedback 센서, 상태 관리 — 이런 인프라 없이 "AI 써봐라"만 하면 혼란만 생긴다.
